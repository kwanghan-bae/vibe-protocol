# ✅ 검증 체크리스트 (Verification Checklists)

---

## 1. 코드 생성 전 체크리스트 (Pre-Generation)

코드를 작성하기 전에 반드시 확인해야 하는 항목들.

### 1.1 요구사항 확인
| # | 항목 | 확인 방법 |
|:---|:---|:---|
| 1 | 사용자 의도를 정확히 파악했는가? | 모호하면 역질문 |
| 2 | 기존 코드와의 관계를 파악했는가? | `rg`, `sg`로 관련 코드 검색 |
| 3 | 영향 범위를 파악했는가? | 호출 관계, 의존성 확인 |

### 1.2 환경 확인
| # | 항목 | 확인 방법 |
|:---|:---|:---|
| 4 | 대상 파일이 존재하는가? | `fd`, `ls`로 확인 |
| 5 | 사용할 라이브러리가 설치되어 있는가? | `build.gradle`, `package.json` 확인 |
| 6 | 설정 파일이 올바른가? | 실제 설정 파일 내용 확인 |

### 1.3 지식 확인
| # | 항목 | 확인 방법 |
|:---|:---|:---|
| 7 | 사용할 API가 실제로 존재하는가? | `google_web_search`로 공식 문서 확인 |
| 8 | 최신 버전에서도 동작하는가? | 버전 호환성 확인 |
| 9 | deprecated된 기능을 사용하려 하지 않는가? | 공식 문서 확인 |

---

## 2. 코드 생성 후 체크리스트 (Post-Generation)

코드를 작성한 후에 반드시 확인해야 하는 항목들.

### 2.1 완전성 확인
| # | 항목 | 자가 점검 |
|:---|:---|:---|
| 1 | import 문이 모두 포함되어 있는가? | ✅/❌ |
| 2 | 모든 함수/메서드가 완전히 구현되었는가? | ✅/❌ |
| 3 | TODO, 생략, 플레이스홀더가 없는가? | ✅/❌ |
| 4 | 에러 처리가 포함되어 있는가? | ✅/❌ |

### 2.2 정확성 확인
| # | 항목 | 자가 점검 |
|:---|:---|:---|
| 5 | 존재하지 않는 API를 호출하지 않는가? | ✅/❌ |
| 6 | 파일 경로가 실제로 존재하는가? | ✅/❌ |
| 7 | 변수명/함수명 오타가 없는가? | ✅/❌ |
| 8 | 타입이 올바른가? | ✅/❌ |

### 2.3 품질 확인
| # | 항목 | 자가 점검 |
|:---|:---|:---|
| 9 | 한국어 주석이 포함되어 있는가? | ✅/❌ |
| 10 | 코드 스타일이 프로젝트 규칙을 따르는가? | ✅/❌ |
| 11 | 불필요한 코드가 없는가? | ✅/❌ |
| 12 | 테스트 코드가 함께 제공되었는가? | ✅/❌ |

## 3. 커밋 전 체크리스트 (Pre-Commit)

커밋을 제안하기 전에 반드시 확인해야 하는 항목들.

### 3.1 빌드 검증
```bash
# Kotlin/Java 프로젝트
./gradlew clean build   # 전체 빌드 + 테스트
./gradlew ktlintCheck   # 코드 스타일 검사

# Node.js 프로젝트
npm run build           # 빌드
npm run test            # 테스트
npm run lint            # 린트

# Python 프로젝트
pytest                  # 테스트
black --check .         # 포맷 검사
mypy .                  # 타입 검사
```

### 3.2 최종 확인
| # | 항목 | 상태 |
|:---|:---|:---|
| 1 | 빌드가 성공하는가? | ✅ 필수 |
| 2 | 모든 테스트가 통과하는가? | ✅ 필수 |
| 3 | 린트 검사가 통과하는가? | ✅ 필수 |
| 4 | 커밋 메시지가 한국어인가? | ✅ 필수 |
| 5 | 컨벤션(feat:, fix: 등)을 따르는가? | ✅ 필수 |

### 3.3 커밋 메시지 템플릿
```
feat: 주문 금액 계산 기능 추가

- OrderCalculator 클래스 구현
- 할인율 적용 로직 포함
- 단위 테스트 2개 추가
```

---

## 4. 검증 실패 시 대응

### 돌드 실패
1. 에러 메시지 정확히 분석
2. 원인 파악 후 수정
3. 다시 빌드하여 확인
4. **빌드 성공할 때까지 커밋 제안하지 않음**

### 테스트 실패
1. 실패한 테스트 케이스 확인
2. 기대값과 실제값 비교
3. 로직 수정 후 재실행
4. **모든 테스트 통과할 때까지 커밋 제안하지 않음**

### 린트 실패
1. 위반 항목 확인
2. 자동 수정 가능하면 `ktlintFormat` 또는 `npm run lint:fix` 실행
3. 수동 수정 필요하면 직접 수정
4. **린트 통과할 때까지 커밋 제안하지 않음**

---

## ⚠️ REMINDER

**검증 없이 커밋을 제안하는 것은 규정 위반이다.**

- 빌드 실패 → 커밋 제안 금지
- 테스트 실패 → 커밋 제안 금지
- 린트 실패 → 커밋 제안 금지
